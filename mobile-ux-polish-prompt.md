# PenMark 移动端 UX 深度优化任务

## 现状

PenMark（知著）是一个微信图文笔记应用。上一轮做了基础的移动端响应式适配：
- 单断点 760px 侧边栏抽屉化
- 编辑器在小屏缩小内边距
- 图片/表格拖拽加了 touch 事件
- 底部工具栏没有，还是桌面端顶部横向滚动

整体体验打分：**能用，但不像一个移动端 App。** 距离飞书文档移动端的品质，还有大量工作。

## 你的任务

把这个应用从"PC 页面缩放版"变成"手机上真正好用的编辑工具"。你需要自己在浏览器里跑起来感受（`node server.js`，端口 3001，先看 PC 端再切移动端模拟器），然后做一次彻底的移动端 UX 升级。

## 核心方向

### 1. 底部固定导航栏（必须）
- 移动端不应该有顶部横向滚动的工具栏
- 做一个固定在底部的导航/操作栏，拇指触达
- 至少包含：切换文档列表（汉堡）、当前文档名、搜索/新建、格式快捷操作
- 参考飞书移动端的底部栏设计

### 2. 手势交互体系（必须）
- **边缘滑动手势**：从左边缘右滑打开侧边栏抽屉，带橡皮筋效果
- **长按上下文菜单**：文档列表项长按弹出操作选项（替代桌面端右键）
- **双击**：双击选中词语（目前移动端选词困难）
- 侧边栏内的滚动不要穿透到背景

### 3. 图片查看器（应该）
- 点击图片全屏预览，暗色背景
- 支持双指缩放和双击缩放
- 多图时左右滑动切换
- 有关闭按钮和图片计数

### 4. 弹窗系统移动端重构（应该）
- 设置面板（目前 720px 固定宽度在小屏溢出）
- 所有弹窗在 ≤760px 改为从底部滑入的 Bottom Sheet
- 支持向下拖拽关闭（超过阈值释放）
- 有圆角和毛玻璃背景

### 5. 动画质感升级（应该）
- 侧边栏打开/关闭用 spring 缓动曲线，不要线性 0.2s
- 按钮点击有水波纹或微缩放反馈
- 列表项插入/删除有过渡动画
- 浮动菜单出现/消失有自然动画

### 6. 键盘适配（应该）
- 移动端键盘弹出时，编辑器可见区域不被遮挡
- 浮动菜单在键盘弹出后重新定位
- iOS 键盘上方可考虑格式快捷栏

### 7. 下拉刷新 + 无限滚动（可选）
- 文档列表下拉刷新
- 文档多时使用虚拟滚动或懒加载

### 8. 触控体验细节（必须）
- 所有交互元素最小触控区 44px（目前工具栏按钮只有 28px）
- 添加 `touch-action: manipulation` 消除 300ms 延迟
- 浮动菜单在移动端改为底部弹出式，不要浮在任意位置
- 移动端浮动菜单按钮要大，图标+文字竖排

## 设计约束

- 不要引入任何框架（React/Vue/jQuery 等）
- 不要改动桌面端行为
- 保持三种主题（纸张/飞书/暗色）正常工作
- 保持现有 CSS 变量体系
- 保持快捷键体系不变
- 安全区域要用 `env(safe-area-inset-*)`

## 期望效果

在 375px 宽度的手机上：
- 打开应用感觉像一个原生 App，不是缩放出来的网页
- 所有操作手指可达，不需要精准点击
- 动画自然流畅，有质感
- 图片可以像微信一样点开看大图、双指缩放
- 侧边栏可以像 iOS 一样从左边缘滑动拉出
