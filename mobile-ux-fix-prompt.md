# PenMark 移动端 UX 修复 + 重设计提示词

## 背景

上一轮做了移动端基础适配（底部导航栏、格式 Sheet、手势、图片查看器、键盘适配）。但有几个方向性问题需要重新设计。

仓库代码在当前 branch 上，`node server.js` 启动后默认端口 3001。先跑起来在移动端模拟器里感受一下现状。

## 三个核心改动

### 1. 底部导航栏重新设计

**现状问题**：现在底部栏是「文档 / 格式 / 新建 / 撤销 / 更多」五个按钮。没人把"格式""撤销"放底部主导航栏的，这和移动端导航的肌肉记忆冲突。

**改法**：参考飞书移动端文档的底部栏设计——
- 左侧：汉堡菜单图标（打开文档列表抽屉）
- 中间：当前文档标题（点按可编辑标题）
- 右侧：搜索 / 新建文档图标
- 不要放格式、撤销这些编辑操作在导航栏
- 编辑操作自然的方式是：选中文字后弹出浮动菜单，或者键盘上方有格式工具栏

### 2. 左滑手势不要和 Android 返回冲突

**现状问题**：从屏幕最左边缘 20px 右滑打开抽屉的手势，和 Android 系统的返回手势（屏幕左边缘右滑 = 返回上一页）完全重叠。用户想返回上一页时会打开侧边栏，想开侧边栏时可能触发系统返回。

**改法**：
- 不要用全屏左边缘滑动手势
- 改用这两种方案之一（或用更合理的方式）：
  - **方案 A**：汉堡菜单按钮打开抽屉（参考大多数 App），不做边缘滑动手势
  - **方案 B**：保持边缘滑动但识别层——只有当侧边栏已打开时，从抽屉区域右滑才关闭；打开只通过按钮触发
  - **方案 C**：只在一小块区域（例如左侧 20px 有一个视觉指示条/把手）才能滑动打开
- 无论哪种方案，不要从整个左边缘全区域响应滑动

### 3. UI 全面现代化

现在的界面比较朴素。请在移动端做一轮视觉升级：

- **圆角和阴影**：统一用更大的圆角（12-16px），阴影用带颜色的多层阴影而非纯黑
- **毛玻璃效果**：侧边栏背景、弹窗遮罩、浮动菜单等用 `backdrop-filter: blur()`
- **间距系统**：段落、列表项、卡片之间保持一致的 12-16px 间距
- **字体层级**：标题 24px、正文 16px、辅助文字 13-14px 的清晰层级
- **图标**：用 feather-icons 风格的 SVG 小图标，统一 20px
- **动画**：所有过渡用 `cubic-bezier(0.4,0,0.2,1)` 或 spring 曲线，250-350ms
- **配色**：在三种主题下都确保移动端的对比度舒适（不要太亮/太暗）
- **空状态**：文档列表为空时给一个美观的空状态插画或占位（而非现在一片空白）
- **触控反馈**：按钮点击有缩放/脉冲反馈（`transform: scale(0.95)`），列表项长按有视觉暗示

### 需要修复的具体 bug

- **格式 Sheet 点空白处关不掉**：`e.target === sheet元素` 太严格，应判断是否在 content 区域外
- **`updateMobileNav()` 不响应 resize**：旋转设备后导航栏状态不更新
- **`iv-counter` 的 safe-area 被覆盖**：双重 `top` 声明，第二个覆盖了第一个
- **长按菜单 touch 坐标**：在 `setTimeout` 回调中访问 `e.touches[0]` 不可靠，应在 `touchstart` 时保存坐标
- **iOS 键盘弹出时**：浮动菜单的文字菜单在 compact 模式下移动端不显示（CSS `display:none`），应该让它显示在键盘上方
- **横屏适配**：底部导航栏在横屏时图标无文字标签，难以辨认

## 设计约束

- 不引入框架
- 桌面端行为不动
- 三种主题都正常工作
- 安全区域用 `env(safe-area-inset-*)`
