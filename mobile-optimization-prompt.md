# PenMark 移动端优化任务

## 项目简介

PenMark（知著）是一个微信图文笔记应用，Node.js + Express 后端，前端是原生 HTML/CSS/JS（无框架）。目前只有桌面端布局，你可以在 `public/` 目录下找到全部前端代码。请你在浏览器里跑起来看看效果，然后做完整的移动端优化。

启动方式：`node server.js`，默认端口 3001。

## 核心目标

把这个应用从"桌面端勉强能用手机打开"变成"手机上的飞书文档级别体验"。

## 参考标杆

飞书文档移动端的设计风格：
- 触控优先，所有操作拇指可达
- 简洁克制的排版，留白充裕
- 卡片化 + 轻微阴影的层次感
- 底部操作栏承载主要编辑功能
- 侧滑抽屉、底部 Sheet 等移动端原生交互模式

## 你需要重构的文件

都放在 `public/` 目录下：
- `styles.css` — 全部样式
- `index.html` — 主应用页
- `app.js` — 主应用逻辑
- `editor.js` — 编辑器核心
- `login.html` + `login.js` — 登录/注册
- `share.html` + `share.js` — 分享查看页

以及项目根目录的 `PenMark.html`（独立单文件版）。

## 方向性要求（不限制具体实现方式）

### 交互层面
- 图片缩放、拖拽位移目前只支持鼠标事件，移动端完全不可用，需要改成触摸手势
- 文档/文件夹的右键菜单在移动端没有替代方案，想一套触控方案（比如长按）
- 文字选区后的浮动格式菜单，移动端要以底部 Sheet 形式呈现
- 侧边栏在手机上做成滑入抽屉，支持手势
- `insertLink` 目前用的是 `window.prompt()`，移动端体验极差，改掉

### 视觉层面
- 所有交互元素触控区域至少 44px，现在很多地方只有 22-28px
- 编辑器在小屏上边距太大，文章根本没几个字就被挤没了
- 考虑使用级联断点而不是只有一个 760px 断点
- iPhone 刘海屏和底部横条不能遮挡内容
- Modal 弹窗在小屏上不要居中浮动了，用底部 Sheet 更自然
- 三种主题（纸张/飞书/暗色）在移动端都要正常

### 架构约束
- 只改移动端行为，桌面端功能一个都别动
- 不要引入任何第三方框架或库（React/Vue/jQuery 等）
- 保持现有的 CSS 变量体系
- 保持现有的主题切换机制

## 期望效果

在 375px 宽度（iPhone SE）上：
- 能像用飞书一样自然地浏览、编辑、管理文档
- 所有操作触手可及，不需要精确的鼠标点击
- 视觉效果干净利落，不像是桌面端缩放出来的
